# 🎬 Find My Movie — агент-консультант по фильмам

Диалоговый ассистент, который подбирает фильмы по смыслу, рассказывает детали,
находит похожее и ищет свежие отзывы в интернете. Работает поверх собственной
базы из **1680 фильмов 2019–2026** (спарсены с [film.ru](https://www.film.ru)).

> Текст исходного задания — в [TASK.md](TASK.md).

## Как это устроено

```
Пользователь
     │
     ▼
  Router (llama-3.1-8b-instant)  ← классифицирует сообщение одним словом
     │
     ├─ smalltalk → прямой ответ головы (без базы и инструментов)
     │
     └─ search    → AgentExecutor (gpt-oss-120b) сам выбирает инструменты:
                        🔍 search_content  — гибридный поиск (семантика + BM25) + фильтры
                        📄 get_details     — полная карточка фильма по названию
                        🎯 find_similar    — похожие фильмы по названию
                        🌐 web_search      — отзывы/новости через Tavily
     │
  Память (ConversationBufferMemory) — общая на оба пути, держит весь диалог
```

Агент сделан на **классическом LangChain** (`AgentExecutor` + tool-calling),
не на LangGraph.

**Ретривер — гибридный.** `search_content` ищет одновременно по смыслу
(плотный e5-large, косинус) и по словам (разреженный BM25 с русским стеммингом),
а Qdrant сливает два рейтинга через RRF (Reciprocal Rank Fusion) в один запрос.
Семантика ловит настроение и описание («что-то напряжённое про космос»), лексика —
точные названия, имена актёров и режиссёров, где чистые эмбеддинги промахиваются.

### Модели (все бесплатные)

| Роль | Модель | Где |
|---|---|---|
| Эмбеддер (семантика) | `intfloat/multilingual-e5-large` (dim 1024) | локально, fastembed — ключ не нужен |
| Лексика (BM25) | `Qdrant/bm25` (sparse, рус. стемминг) | локально, fastembed — ключ не нужен |
| Голова (ответы) | `openai/gpt-oss-120b` | Groq API (бесплатный ключ) |
| Роутер (классификация) | `llama-3.1-8b-instant` | Groq API |
| Веб-поиск | Tavily | Tavily API (бесплатный ключ) |
| Векторная база | Qdrant (локальный файловый режим) | `data/db/` |

## Структура проекта

```
.
├── data/
│   ├── raw.csv          # собранные фильмы (1680 шт)
│   └── db/              # локальная база Qdrant (векторы + метаданные)
├── parser.py            # парсер film.ru (2 фазы, возобновляемый)
├── embeddings.py        # общий слой эмбеддера e5-large
├── indexer.py           # индексация raw.csv -> Qdrant
├── tools.py             # 4 инструмента агента поверх Qdrant + Tavily
├── agent.py             # роутер + агент + память (классический LangChain)
├── app.py               # запуск: чат в терминале
├── ui_streamlit.py      # запуск: веб-интерфейс (Streamlit, мультичат)
├── bot.py               # запуск: Telegram-бот
├── pyproject.toml       # зависимости (uv)
└── .env.example         # шаблон ключей
```

## Запуск

Нужен [uv](https://docs.astral.sh/uv/) и Python 3.13.

### 1. Установить зависимости

```bash
uv sync
```

### 2. Прописать ключи

```bash
cp .env.example .env
```

Заполни в `.env`:
- `GROQ_API_KEY` — бесплатный ключ на https://console.groq.com/keys (**обязательно**)
- `TAVILY_API_KEY` — бесплатный ключ на https://app.tavily.com (для `web_search`)
- `TELEGRAM_BOT_TOKEN` — токен от [@BotFather](https://t.me/BotFather) (только для бота)

Эмбеддер локальный — ключ для него не нужен.

### 3. Собрать базу

Датасет `data/raw.csv` уже в репозитории. Достаточно проиндексировать его в Qdrant:

```bash
uv run python indexer.py          # построить базу (data/db/)
uv run python indexer.py --check  # проверить, что в базе, без записи
```

> Заново спарсить данные (не обязательно): `uv run python parser.py`

> ⚡ **Индексация на GPU (NVIDIA/CUDA).** Эмбеддинг 8142 фильмов на CPU — долго
> (~1.5 ч). На видеокарте с CUDA быстрее в разы: поставь GPU-сборку fastembed —
> `uv pip install fastembed-gpu` (нужен установленный CUDA/cuDNN) — она сама
> задействует `CUDAExecutionProvider`. На Apple (MPS) ускорения нет: fastembed
> работает на ONNX Runtime, а MPS — это PyTorch.

> 🔗 Если кто-то уже собрал базу — можно не индексировать заново, а просто
> получить готовую папку `data/db/` (эмбеддинг детерминирован, результат тот же).

### 4. Запустить агента — на выбор

```bash
uv run python app.py   # чат в терминале
uv run streamlit run ui_streamlit.py   # веб-интерфейс (откроется в браузере, :8501)
uv run python bot.py   # Telegram-бот
```

> ⚠️ Запускать можно **только что-то одно за раз** — Qdrant в файловом режиме
> блокирует базу на один процесс.

## Что попробовать

- «посоветуй напряжённый фильм про выживание в космосе» — семантический поиск
- «расскажи подробнее про первый» — детали + память
- «а что похожее посоветуешь?» — похожие фильмы
- «что критики пишут про …» — веб-поиск отзывов
- «привет, как дела?» — обычная болтовня (без поиска)

## Командная сборка базы (объединение датасетов)

Каждый участник парсит свой период → отдаёт `data/raw_<годы>.csv`. Один человек
склеивает их (дедуп по `url`) в общий `data/raw.csv` и прогоняет `indexer.py`.
Эмбеддинг детерминирован, поэтому собранную базу можно как раздать готовой
(`data/db/`), так и пересобрать у себя из общего CSV.
