# Проектное задание: Find My Book / Movie / Recipe

## Выбери один вариант

| Вариант | Домен | Источник для парсинга |
|---|---|---|
| 📚 **Find My Book** | Книги | [biblio-globus.ru](https://www.biblio-globus.ru/catalog) |
| 🎬 **Find My Movie** | Фильмы | [film.ru](https://www.film.ru/movies) или [kinorium.com](https://kinorium.com/films/) |
| 🍳 **Find My Recipe** | Рецепты | [povarenok.ru](https://www.povarenok.ru/recipes/) |

Все три варианта реализуются по одной архитектуре — разница только в домене и метаданных.

---

## Часть 1. Сбор данных

Напиши парсер, который собирает **не менее 500 позиций** из выбранного источника.

### Что собирать

**📚 Книги** (`books.csv`):
```
title, author, year, genres, description, rating, num_ratings, pages, publisher, cover_url, url
```

**🎬 Фильмы** (`movies.csv`):
```
title, original_title, year, genres, director, cast, country, rating, num_ratings, duration_min, description, poster_url, url
```

**🍳 Рецепты** (`recipes.csv`):
```
title, category, cuisine, description, ingredients, cooking_time_min, servings, difficulty, rating, num_ratings, author, image_url, url
```

### Требования к парсеру

- Используй `requests` + `BeautifulSoup` (или `httpx` + `selectolax`)
- Добавь задержки между запросами (`time.sleep`) — не клади сайт
- Сохраняй прогресс: если прервался, должен продолжить с места остановки
- Логируй ошибки, не падай на битых страницах
- Итоговый файл — `data/raw.csv`

## Часть 2. Агент-консультант

Твой агент — это **полноценный консультант**, который ведёт диалог и помогает найти идеальную книгу, фильм или рецепт.

### Что должен уметь агент

**Свободный диалог**
- Поддерживать обычный разговор без поиска: _"привет"_, _"что посоветуешь?"_, _"я вчера приготовил борщ, получилось отлично"_
- Не лезть в базу на small talk — отвечать естественно и по-человечески
- Плавно возвращать разговор к теме: если пользователь просто болтает, агент ненавязчиво предлагает помощь с поиском

**Поиск и рекомендации**
- Находить контент по смысловому описанию: _"хочу что-то про одиночество в мегаполисе"_, _"быстрый ужин из курицы без духовки"_
- Фильтровать по параметрам: жанр, год, рейтинг, время приготовления, сложность, кухня
- Рекомендовать похожее: _"понравился Дюна, что ещё посмотреть?"_, _"хочу что-то похожее на этот рецепт"_
- Отвечать на запросы по настроению или ситуации: _"хочу лёгкое и смешное на вечер"_, _"что приготовить на большую компанию?"_

**Работа с метаданными**
- Показывать полную карточку по запросу: _"расскажи подробнее про первый результат"_
- Отвечать на уточняющие вопросы: _"а сколько там минут готовить?"_, _"кто режиссёр?"_, _"какие ингредиенты нужны?"_
- Сравнивать несколько позиций между собой

**Контекст и память**
- Помнить всё что обсудили в рамках диалога
- Учитывать отказы: если пользователь сказал _"это не моё"_ — не предлагать похожее
- Накапливать профиль предпочтений: _"ты уже сказал что не любишь хорроры"_, _"ты вегетарианец — я это учту"_

**Свежая информация**
- Искать отзывы и рецензии в интернете по запросу
- Отвечать на актуальные вопросы: _"вышел ли новый фильм этого режиссёра?"_, _"есть ли веганская версия этого блюда?"_

### Инструменты агента (минимум 3)

| Инструмент | Что делает |
|---|---|
| `search_content` | Семантический поиск по базе (+ опциональная фильтрация) |
| `get_details` | Полная карточка по ID из базы |
| `find_similar` | Похожий контент по ID |
| `web_search` | Поиск отзывов и свежей информации через Tavily |

### Архитектура

```
Пользователь
     │
     ▼
  Router (small talk / поиск / детали)
     │
     ├─► RAG Chain + Memory      ← семантический поиск по Qdrant
     │
     └─► LangGraph Agent         ← многошаговые запросы с tool calls
              │
              ├─ search_content
              ├─ get_details
              ├─ find_similar
              └─ web_search
```

Используй паттерны из занятий:
- **Индексация и поиск** → ноутбук 01_01
- **RAG + Memory + Router** → ноутбук 01_02
- **Tools + LangGraph** → ноутбук 01_03

## Критерии оценки

| Блок | Что проверяем |
|---|---|
| Парсер | Корректно собирает ≥ 500 позиций, обрабатывает ошибки, не дублирует записи |
| Индексация | Грамотно формирует текст для эмбеддинга, метаданные доступны для фильтрации |
| Поиск | Семантика работает (нерелевантные результаты — минус), фильтры применяются |
| Агент | Минимум 3 инструмента, агент выбирает нужный без подсказок |
| Диалог | Память работает, агент удерживает контекст минимум на 5 сообщений |
| Веб-поиск | Tavily подключён и вызывается в нужных случаях (не всегда, не никогда) |
| Качество ответов | Агент не выдумывает — только из базы или из веб-поиска |
